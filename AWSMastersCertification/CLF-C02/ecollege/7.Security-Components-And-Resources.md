# Identify Components and Resources for Security

### Introduction to AWS Security Capabilities
AWS provides built-in security features, tools, and documentation to help customers protect their cloud workloads. Security is AWS is based on the AWS Shared Responsibility Model.

#### Key security components include:
* **AWS Security Services** (e.g. AWS WAF, Security Groups, AWS Shield)
* **AWS Security Documentation and Resources**
* **Third-party Security Solutions** (AWS Marketplace)
* **AWS Security Monitoring and Best Practices**

<br>

### Scurity Capabilities AWS Provides
AWS offers multiple security services of network security, threat detection, compliance monitoring, and access control

#### AWS Security Features and Services
| **Security Feature/Service**           | **Purpose**                                                                             |
|----------------------------------------|-----------------------------------------------------------------------------------------|
| **Security Groups**                    | Act as a firewall for EC2 instances, controlling inbound/outbound traffic               |
| **Network ACLs**                       | Ad additional layer of security at the subnet level, controlling traffic rules          |
| **AWS Web Application Firewall**       | Protects web applications from SQL injection, XSS, and other threats                    |
| **AWS Shield**                         | DDoS protection for AWS services (Standard is free, Advanced is paid)                   |
| **AWS Key Management Service**         | Manages encryption keys for data security                                               |
| **AWS Identity and Access Management** | Controls user access and permissions for AWS resources                                  |
| **AWS Secrets Manager**                | Stores and manages sensitive information (e.g., passwords, API keys)                    |
| **Amazon Macie**                       | Uses AI to detect sensitive data (e.g. PII) in Amazon S3                                |
| **Amazon GuardDuty**                   | Threat detection service using machine learning to identify risks                       |
| **AWS Config**                         | Monitors AWS resource configurations for compliance                                     |
| **AWS Security Hub**                   | Centralised security dashboard integrating security findings from multiple AWS services |

<br>

### Security-Related Documentation Provided by AWS
AWS provides official security documentation, blogs and whitepapers to help customers implement best security practices

#### Where to find AWS Security Information

| **AWS Security Resource** | **Description** |
|------|------|
| **AWS Knowledge Center** | Contains FAQs and troubleshooting guides |
| **AWS Security Center** | Security best practices, compliance programs, and incident response guidance |
| **AWS Security Blog** | Regular updates on AWS security features and recommendations |
| **AWS Well-Architectured Framework - Security Pillar** | Best practices for security in cloud architecture |
| **AWS Compliance Center** | Details on global compliance standards (SOC, PCIDSS, GDPR, HIPAA) |
